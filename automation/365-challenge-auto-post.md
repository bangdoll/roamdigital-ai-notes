這份教學將協助你使用 `pub365` 指令，一鍵將你的攝影挑戰日記與照片發布到 [rd.coach](https://rd.coach)。

## 🎯 核心概念
你只需要專注於 **寫日記** 和 **存照片**，剩下的（上傳圖片、設定分類、產生網址、發布文章）全交給程式處理。

---

## 🚀 準備工作 (SOP)

在執行指令前，請確保完成以下兩件事：

### 1. 照片歸位 📸
將當天的照片放入資料夾：
`~/Projects/00.AI-Notes_Local/365攝影挑戰/365攝影挑戰Photos 2026/`

*   **檔名格式**：`365攝影挑戰YYYYMMDD.JPG` (或 `.jpg`)
*   **範例**：`365攝影挑戰20260214.JPG`

### 2. 日記寫好 📝
在 Markdown 檔案中寫好當天的日記：
`~/Projects/00.AI-Notes_Local/365攝影挑戰/365攝影挑戰/365攝影挑戰202602.md`

*   **格式要求**：標題必須包含 `#365攝影挑戰 20260214...`
*   程式會自動抓取該標題下、直到下一個 `---` 分隔線之間的內容。

---

## ⚡️ 開始發布

打開終端機 (Terminal)，輸入以下指令：

### 發布「今天」的挑戰
```bash
pub365
```
*這會自動抓取今天的日期，尋找對應的日記與照片。*

### 補發「特定日期」的挑戰
例如補發 2026 年 2 月 12 日的：
```bash
pub365 --date 20260212
```

---

## ✅ 程式會幫你做什麼？

1.  **解析日記**：從 Markdown 檔案中精準提取當天的標題與內文。
2.  **上傳照片**：將照片上傳至 WordPress 媒體庫，並以此作為「精選圖片」。
    *   *自動處理檔名，避免中文亂碼問題。*
3.  **設定分類**：自動勾選「365challenge」分類。
4.  **生成網址**：自動產生符合 SEO 的英文網址。
    *   格式：`365-20260214-045-365-day3698` (無中文，對 Google 友善)
5.  **正式發布**：文章直接上線，你馬上就能在網站看到。

## ❓ 常見問題

**Q: 如果照片忘記放怎麼辦？**
A: 程式會發出警告 `⚠️ No image uploaded`，文章還是會發布，但就沒有精選圖片。你可以事後手動去後台補，或是刪掉文章重發。

**Q: 若文章已經發布過，再執行一次會怎樣？**
A: 程式會偵測標題，如果標題完全一樣，它會跳過 (`Skipping`)，防止重複發布。

**Q: 網址有一堆亂碼？**
A: 請確認你使用的是新版腳本（已更新支援英文 Slug）。舊文章若已發布，需手動在後台修改網址。
